
# 📱 3D Match Pro：推荐查询功能设计 (便利美观版)


**目标要求，实现原理

例如为https://ds.3ddl.net/index.php?ctl=Goods_Goods&met=goods&type=goods&gid=51448
其中，链接里面的gid就是我需要输入的id，
我需要你获得这个连接下页面里面的推荐指数的数据，具体为，总的推荐指数为多少，普通会员的推荐是多少，专家推荐是多少，数字认证的推荐是多少。
然后制作成一个接口，加入软件中，这样当我更换gid时，可以直接查询，具体ui方案如下。

## 1. 核心交互流程：一键即得 (Easy-Flow)

为了保证极致便利，查询功能的交互设计如下：

1. **输入触发**：用户在首页 `Goods ID` 输入框输入数字。
2. **点击查询**：点击输入框末端的 **[🔍 极光图标]**。
3. **动态展示**：
* 按钮瞬间变成一个微小的 **[旋转加载动效]**。
* 查询完成后，在输入框下方**平滑滑出**一个美观的悬浮信息条（Toast 升级版）。


4. **自动同步**：查询到的赞数会自动显示在首页的“当前进度”中，作为任务的起始参考。

---

## 2. 界面设计方案 (视觉美观)

### 2.1 首页组件优化

我们对原有的 `Dashboard` 进行微调，加入一个名为 **"Quick Info"** 的视觉槽位。

* **搜索框设计**：输入框采用圆角矩形，右侧集成一个带有渐变色的查询按钮。
* **信息展示条 (Result Strip)**：
* **材质**：半透明毛玻璃材质 (Backdrop Filter)。
* **内容**：左侧显示商品的小缩略图，中间是标题，右侧是加粗的 **[获赞数]**。
* **动效**：使用“弹性展开”动画（Spring Animation），数据出现时数字有滚动跳变效果。

## 3. 功能设计文档 (详细版)

### 3.1 功能定义

* **功能名称**：推荐看板 (Instant Popularity Check)




### 3.2 详细设计规范

| 维度 | 设计细节 | 实现建议 (Android Studio) |
| --- | --- | --- |
| **便利性** | **自动联想**：输入 6 位 ID 后图标高亮提醒可查询。 | 使用 `TextWatcher` 监听输入长度。 |
| **美观度** | **色彩反馈**：查询成功为“极光青”，ID 错误为“珊瑚粉”。 | 使用 `AnimatedVectorDrawable` 切换颜色。 |
| **反馈感** | **数字滚动**：数字不是生硬跳出，而是像里程表一样滚动。 | 使用 `TextView` 的 `animateText` 扩展。 |
| **容错性** | **断网处理**：网络不可用时，查询按钮变灰并显示微弱的震动。 | 结合 `ConnectivityManager` 预判网络状态。 |

## 4. 关键代码参考 (Kotlin)

在 Android Studio 中，你可以通过一个简单的扩展函数来实现这个漂亮的查询逻辑：

```kotlin
// 查询按钮点击事件
binding.btnQuery.setOnClickListener {
    val id = binding.etGoodsId.text.toString()
    
    // 执行漂亮的小动画
    it.startLoadingAnimation() 
    
    lifecycleScope.launch {
        val result = repository.getLikeCount(id)
        
        result.onSuccess { data ->
            // 平滑展开结果条
            binding.resultStrip.apply {
                visibility = View.VISIBLE
                textName.text = data.name
                textCount.animateToNumber(data.likes) // 数字滚动动画
                alpha = 0f
                animate().alpha(1f).setDuration(300).start()
            }
        }.onFailure {
            showErrorTips("未找到商品，请检查ID")
        }
    }
}

```

## 5. 总结：为什么这样设计？

* **不复杂**：没有增加多余的页面，只是在现有 ID 输入框上做了增强。
* **很方便**：输入完顺手一点就能看结果，不需要去浏览器查。
* **很漂亮**：通过毛玻璃效果和数字滚动动画，让一个枯燥的查询请求变得具有“仪式感”。
