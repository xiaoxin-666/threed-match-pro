# 📱 3D Match Pro (3D 大赛助手) 功能设计文档

## 1. 项目定位

本项目旨在将原有的自动化脚本升级为成熟的 Android 原生应用。通过原生并发处理、数据持久化及更友好的 UI 交互，为用户提供高效、稳定、安全的商品人气管理方案。

---

## 2. 核心功能架构

App 将划分为四个核心维度：**执行引擎、任务管理、安全策略**。

### 2.1 自动化执行引擎

* **多线程并发**：利用 Kotlin 协程 (Coroutines) 实现多任务并行处理，互不阻塞。
* **网络协议栈**：使用 OkHttp 4 进行底层通信，支持自定义 Header、Cookie 及长连接优化。
* **异常熔断**：当检测到服务器返回“操作频繁”或网络 404 时，自动触发暂停保护。

### 2.2 任务管理系统

* **任务持久化**：使用 Room 数据库存储商品 ID、目标次数及执行记录。
* **队列模式**：支持“顺序执行”或“同步执行”模式切换。
* **进度快照**：App 意外关闭后，重启可从上次断点处继续执行。



### 2.4 安全与仿生策略

* **动态 User-Agent**：内置主流手机型号 UA 池，每次请求随机切换。
* **抖动延迟**：在固定间隔基础上增加 $\pm 20\%$ 的随机时间，模拟真人操作频率。

## 3. 页面设计方案 (UI/UX)

### 3.1 首页：控制面板 (Dashboard)

* **视觉重点**：顶部的“总任务进度”环形图。
* **功能布局**：
* **状态卡片**：显示今日累计点赞数、当前运行中的任务数。
* **快速操作**：一键停止所有任务、一键清除成功日志。
* **底部浮标**：快速新建任务。



### 3.2 任务页：列表与配置 (Task Manager)

* **任务列表**：以卡片形式展示每个商品 ID。卡片包含：商品名（抓取）、ID、完成进度条、状态标签（执行中/已停止/完成）。
* **编辑弹窗**：
* `TextField`：商品 ID 输入。
* `Slider`：频率调节（0.2s - 10s）。
* `Switch`：是否开启“极速模式”。




### 3.4 日志页：控制台 (Live Console)

* **设计风格**：深色背景，模仿代码编辑器。
* **彩色输出**：
* `[INFO]` 蓝色：普通流程。
* `[SUCCESS]` 绿色：成功响应。
* `[ERROR]` 红色：请求失败或服务器报错。

## 4. 技术栈推荐 (Android Studio)

| 维度 | 推荐方案 | 说明 |
| --- | --- | --- |
| **开发语言** | Kotlin | 原生开发首选，代码简洁，协程支持完美。 |
| **UI 框架** | Jetpack Compose | 声明式 UI，开发速度比 XML 快，效果更现代。 |
| **网络请求** | Retrofit 2 + OkHttp | 行业标准，处理 POST 请求和 JSON 解析最稳。 |
| **数据库** | Room (SQLite) | 官方推荐的 ORM 框架，用于保存任务和历史。 |
| **后台运行** | WorkManager / Foreground Service | 保证 App 在后台或锁屏时任务不被系统挂起。 |

---

## 5. 开发建议与避坑

1. **UI 性能优化**：由于日志刷新极快，不要使用简单的 `TextView.append()`，在 Android Studio 中应使用 `RecyclerView` 配合 `DiffUtil`，否则在发送几千次后界面会严重卡顿。
2. **HTTPS 证书**：目标网站是 HTTPS，Android 默认对证书要求严格，建议在 `network_security_config.xml` 中做好配置。
