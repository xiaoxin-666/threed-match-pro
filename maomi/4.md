

## 💎 新功能设计文档：高频并发探测器 (High-Speed Probe)

### 1. 功能概述

* **功能名称**：并发状态探测器。
* **核心目标**：模拟特定浏览器环境，在短时间内并发发送 1-5 个探测请求，并实时统计服务器返回的 `success` 响应数量。
* **应用场景**：用于快速测试特定商品 ID 的响应有效性及服务器并发承载状态。

---

### 2. 页面交互流程 (新页面设计)

新页面将通过首页的“高级功能”入口进入，采用**沉浸式表单**设计：

1. **参数配置**：
* **商品 ID**：默认继承首页输入，支持手动修改。
* **并发数选择**：采用 **分段选择器 (Segmented Control)**，可选 1-5。


2. **动作触发**：点击底部的“执行探测”大按钮，按钮伴随微光进度条动画。
3. **实时统计反馈**：
* 屏幕中心出现一个**数据看板**，动态显示：`已发送: N` / `成功: M`。
* 探测结束后，弹出一个精美的**结果卡片**。


4. **详细审计**：下方保留一个微型日志窗，仅记录每次并发的响应 `msg` 内容。

### 4. UI 页面视觉规划 (Android Studio)

#### A. 探测面板 (Probe Panel)

* **顶部**：显示商品预览（如果已查询）及大的实时成功率百分比。
* **中部**：
* `EditText`：商品 ID（带一键清除图标）。
* `Slider`：并发请求数设置 (1-5)。


* **底部**：巨大的渐变色按钮“开始并发探测”。

#### B. 结果反馈 (Result Feedback)

* 成功次数显示为**翡翠绿**，失败次数为**珊瑚粉**。
* 通过图表（如简单的条形图）展示 5 次请求的成功比例。

### 5. 关键业务代码逻辑 (Kotlin 伪代码)

在 Android Studio 中，你可以利用 `async` 模式并行启动请求：

```kotlin
// 并发探测核心逻辑
fun executeProbe(goodsId: String, threadCount: Int) {
    var successCount = 0
    val jobs = List(threadCount) {
        lifecycleScope.async(Dispatchers.IO) {
            val response = apiService.simulateDetailedPost(goodsId) // 严格按照图片配置Header
            if (response.body()?.msg == "success") {
                synchronized(this) { successCount++ }
            }
        }
    }
    
    lifecycleScope.launch {
        jobs.awaitAll() // 等待所有请求完成
        showResultDialog("探测结束", "成功数: $successCount / $threadCount")
    }
}

```

---

### 6. 为什么这样优化？

* **便利性**：将繁琐的 Header 配置硬编码在底层，用户只需输入 ID 并点击一次，极大地减少了操作负担。
* **漂亮感**：通过数据看板和成功/失败的色彩对比，将原本枯燥的 JSON 数据转化为直观的可视化结果。
* **稳定性**：引入并发统计机制，比单次发送更具参考价值，能有效过滤网络抖动产生的误差。
